Honkit

Honkit

  • Honkitは、GitBookのフォークであり、静的サイトジェネレーターとして使用される
  • GitBookは有料で登録が必要なのでこちらの方が楽?

使い方

  1. Honkitのインストール

  2. まず、Node.jsがインストールされていることを確認してください。その後、以下のコマンドを実行してHonkitをインストールする

    npm install honkit --save-dev

  3. プロジェクトのセットアップ

  4. 新しいプロジェクトディレクトリを作成し、その中に移動

    mkdir mybook
cd mybook

  5. 初期設定ファイルの作成

  6. Honkitの初期設定ファイルを作成する
    npx honkit init

  7. このコマンドを実行すると、README.mdとSUMMARY.mdという2つのファイルが生成される

    • README.md: 本の紹介や概要を書くファイル
    • SUMMARY.md: 本の目次を定義するファイル

  8. 本の内容を追加

  9. README.mdとSUMMARY.mdを編集して、本の内容を追加する

  10. 例えばSUMMARY.mdは以下のようになる

    • フォルダを分けない場合 ```markdown

      Summary

    • Introduction
    • Part1
    • Part2 ```
    • フォルダを分ける場合 ```markdown

      Summary

    • Introduction
    • Chapter 1
      • Contents of Chapter 1
    • Chapter 2
      • Contents of Chapter 2 ```

  11. ローカルサーバーでプレビュー

  12. 以下のコマンドを実行して、ローカルサーバーで本をプレビューする
    npx honkit serve
  13. ブラウザで http://localhost:4000 にアクセスすると本の内容を確認できる

プラグインのインストール

  • https://www.npmjs.com/ から検索するとよい
  • インストールコマンドも同時に表示されるのでそれを使う
  • honkit用とgitbook用がある。どちらで動くかはあまりよくわからない。

book.jsonの設定

  • htmlの設定はbook.jsonで設定する

コードハイライト

  • ハイライトを変えたい場合はprismなどを使う: {"plugins": ["prism", "-highlight"]}

"Honkitで公開"の削除

  • サイドバーにあるHonkitで公開 (Published with Honkit)を削除: {"plugins": ["hide-published-with"]}

数式

  • mathjax: {"plugins": ["mathjax"]}
  • katex: {"plugins": ["katex"]}
  • katex + mhchem: {"plugins: ["katex-mhchem"]}

webで公開

  1. プロジェクトの初期化

  2. ローカルでプロジェクトディレクトリを作成し、Gitリポジトリを初期化する

    mkdir mybook
cd mybook
git init

  3. Honkitプロジェクトのセットアップ

  4. Honkitプロジェクトをセットアップし、HTMLをビルドする

    honkit init
honkit build . docs

  5. ファイルをコミット

  6. すべてのファイルをGitに追加し、コミットする

    git add .
git commit -m "Initial commit"

  7. リモートリポジトリにプッシュ

  8. リモートリポジトリを追加し、ローカルの変更をプッシュする

    git remote add origin https://github.com/your_name/mybook.git
git push -u origin main

  9. GitHub Pagesの設定

  10. GitHubリポジトリの設定ページに移動し、GitHub Pagesのセクションを見つける

  11. "Source"を"main"の"/docs"ディレクトリに設定する

  12. 公開URLの確認

  13. 設定が完了すると、GitHub PagesのURLが表示される
  14. このURLにアクセスすると、docsディレクトリ内のHTMLが公開されていることを確認できる
  15. "About"のWebsiteにGitHub PagesのURLを使うとよい
